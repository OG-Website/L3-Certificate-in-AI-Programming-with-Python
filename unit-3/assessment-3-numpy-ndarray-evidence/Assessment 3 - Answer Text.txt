Q1. Give a general description of the use of NumPy

NumPy is a third-party Python library used for numerical computing and array-based data work. Its full name is Numerical Python, and it is one of the main libraries used when Python needs to handle numbers in a structured and efficient way. Standard Python can store numbers in lists, but lists are general-purpose objects. NumPy is designed specifically for numerical data, so it gives Python a much stronger way to store, calculate, reshape, save and reuse arrays of values.

The main object used in NumPy is the ndarray, which means n-dimensional array. An ndarray can store data in one dimension, like a single list of numbers, two dimensions, like a table of rows and columns, or more dimensions for more complex data. This makes NumPy useful for many types of programming work, including data analysis, scientific computing, artificial intelligence, machine learning, statistics, image processing and engineering calculations. In Unit 3, the ndarray is important because the assessment asks me to create and save one using NumPy in Jupyter Notebook.

NumPy is used when data needs to be handled as a set of values rather than as separate individual variables. For example, a learner could store test scores, prices, measurements, pixel values, sensor readings or statistical data inside an ndarray. Once the values are in an ndarray, NumPy can perform operations across the whole array. This is more practical than writing repeated code for every value. If two arrays have matching shapes, NumPy can add, subtract, multiply or divide their values element by element. This makes numerical work shorter, clearer and easier to check.

Another use of NumPy is inspecting the structure of data. An ndarray has properties such as shape, ndim, size and dtype. Shape shows the rows and columns or wider structure of the array. ndim shows how many dimensions the array has. Size shows the total number of values stored in it. dtype shows the data type, such as integer or floating-point number. These properties are useful because numerical programs need to know what kind of data they are processing. In my Assessment 3 notebook, I use these properties to prove that the ndarray has been created correctly.

NumPy is also used for preparing data before it is used by other libraries. Pandas, for example, can work with NumPy arrays and often uses NumPy internally. This means NumPy is not just an isolated library. It supports wider Python data workflows. In Unit 3, NumPy links naturally to the later Pandas work because NumPy handles numerical arrays while Pandas handles labelled table data. A NumPy ndarray can be used as a starting point before converting or analysing the data in another format.

NumPy is also useful because arrays can be saved and loaded again. A program can create an ndarray, process it, and then save it to a file. NumPy provides different saving methods. np.savetxt() can save the array as readable text or CSV-style data, while np.save() stores the array in NumPy's own .npy format. The .npy format is useful because it preserves the array structure and data type. Loading the file back later proves that the data was not just temporary screen output.

Q2. Describe two or more benefits of using NumPy (3.2).

Benefit 1 - Faster numerical processing

One benefit of using NumPy is efficiency. NumPy arrays are designed for numerical processing, so they are usually faster and more suitable for calculations than standard Python lists. A normal Python list can store mixed types of objects, but that flexibility also means it is not specialised for mathematical operations. A NumPy ndarray stores data in a more regular structure and can apply operations across the whole array. For example, if I have two arrays with matching shapes, NumPy can add the values together element by element without me writing a manual loop for each position. This makes numerical code shorter, clearer and easier to read. In the Assessment 3 notebook, the ndarray stores the score values together in one structure, which means the data can be checked, saved and loaded as one array rather than managed as separate variables.

Benefit 2 - Clear array structure and useful properties

A second benefit is that NumPy gives data a clear structure. An ndarray has properties such as shape, ndim, size and dtype. These help me understand what data I am working with before I process or save it. Shape tells me the layout of the data, such as two rows and three columns. ndim tells me whether the array is one-dimensional, two-dimensional or higher-dimensional. Size tells me the total number of values, and dtype tells me the data type. This is useful evidence in Jupyter Notebook because I can print these properties and show that the array has been created correctly. It also helps prevent mistakes because I can check the array structure before using it in calculations or saving it to files.

Benefit 3 - Saving, reusing and sharing data

A third benefit is that NumPy supports saving and reusing numerical data. With np.savetxt(), an ndarray can be saved as a readable text or CSV-style file. With np.save(), the same ndarray can be saved in NumPy's own .npy format. This is helpful because the data does not disappear when the notebook closes.
