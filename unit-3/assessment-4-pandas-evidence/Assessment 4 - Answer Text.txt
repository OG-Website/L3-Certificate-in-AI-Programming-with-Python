Q1. What is Pandas used for, and outline the difference between Pandas data structures and NumPy ndarrays (4.1, 4.2).

Pandas is used for...(4.1)

Pandas is used for working with structured data in Python. It is mainly used for data analysis, data cleaning, data manipulation and preparing data so it can be understood or used by other libraries. Pandas is useful when data is arranged like a table, because it provides labelled rows and columns instead of leaving the programmer to manage every value manually. It can read and write common file types such as CSV, Excel, JSON and SQL results, which makes it practical for real data work rather than only small examples typed into code.

In my Unit 3 work, Pandas would be used after the earlier NumPy evidence because it gives the data more meaning. NumPy is strong for numerical arrays, but Pandas is better when the values need labels, headings and table-style handling. For example, the Assessment 4 notebook creates an OG Web.site evidence table with tasks, library names, status values and scores. Pandas allows that data to be stored as a DataFrame, printed clearly, accessed by column or row, and changed using commands such as drop(). This is the kind of structure that would be useful when recording assessment evidence, tracking files, checking statuses or preparing tabular data for later analysis.

Pandas is also used for cleaning and checking data. It can handle missing values, remove duplicate rows, filter records, sort data, group data, calculate summary statistics and combine datasets. This makes it useful in AI programming because clean and organised data is needed before machine learning or visualisation work can be reliable. Pandas also works well in Jupyter Notebook because the code, output and explanation can be shown together as evidence.

Difference between Pandas data structures (i.e. Series and DataFrame) and NumPy ndarray...(4.2)

The main difference between Pandas data structures and NumPy ndarrays is the way the data is organised and labelled. A NumPy ndarray is an n-dimensional array that is usually best for numerical calculations. It stores values in a regular grid and is normally homogeneous, meaning the values are of the same data type. This makes NumPy efficient for mathematical work, array operations and saving numerical data. In Assessment 3, I used an ndarray to store score values, check its shape and save it as .txt, .csv and .npy files.

Pandas uses two main data structures: Series and DataFrame. A Series is one-dimensional and works like a labelled column of data. A DataFrame is two-dimensional and works like a table made from columns. The labels are important because values can be accessed by meaningful names instead of only by numerical position. In the Assessment 4 notebook, the Series uses labels such as Planning, Build and Testing, and the DataFrame uses columns such as Task, Library, Status and Score. That makes the output easier to understand in an evidence pack because the meaning of each value is visible.

Another difference is that Pandas can handle mixed data types more naturally. A DataFrame can contain text columns, number columns and status columns in the same table. A NumPy ndarray is better when the data is mainly numerical and arranged in a consistent shape. Pandas is therefore stronger for data cleaning, filtering and analysis, while NumPy is stronger for lower-level numerical processing. The two libraries are often used together: NumPy provides efficient array handling, and Pandas builds labelled analysis tools on top of that style of data.
