Q1. Give a general description of the purpose of Matplotlib and Seaborn, and describe their features (5.1, 5.2).

Matplotlib

The purpose...

Matplotlib is a Python library used to create data visualisations such as line graphs, bar charts, scatter plots, histograms and other figures. Its main purpose is to turn numerical or structured data into a visual format so patterns, comparisons, trends and unusual values can be understood more easily. In AI programming, this is useful because data often needs to be explored before it is used for modelling or decision-making. A table of numbers may be difficult to interpret quickly, but a chart can show whether values are increasing, decreasing, grouped together or spread out.

Matplotlib is also useful because it works well with the other libraries used in Unit 3. NumPy can store numerical arrays, Pandas can organise labelled data into Series and DataFrames, and Matplotlib can display that data visually. For example, after creating a Pandas DataFrame of assessment evidence or scores, Matplotlib could be used to create a bar chart showing the values by category. This makes Matplotlib part of a practical Python data workflow, not just a drawing tool.

Features...

One important feature of Matplotlib is that it gives detailed control over how a visualisation looks. A programmer can set the chart title, x-axis label, y-axis label, legend, colours, marker styles, line styles, figure size and axis limits. This is useful when the chart needs to be clear enough for evidence, reporting or presentation. Matplotlib can also export charts to file formats such as PNG and PDF, which means a finished visualisation can be saved and included in a document.

Another feature is the wide range of plot types. Matplotlib can create line plots for trends, bar charts for category comparisons, scatter plots for relationships, histograms for distributions and many other chart types. It also works inside Jupyter Notebook, so the code and chart output can appear together. This is useful for college evidence because the assessor can see the code used to create the chart and the visual result underneath it. Matplotlib is therefore flexible, detailed and suitable for both quick exploration and more controlled final charts.

Seaborn

The purpose...

Seaborn is a Python data visualisation library built on top of Matplotlib. Its purpose is to make statistical visualisations easier to create and easier to read. It is especially useful when working with Pandas DataFrames because Seaborn can use column names directly when creating charts. This makes it practical for exploring structured data where each column has a meaning, such as category, score, status, time or measurement.

Seaborn is often used when the programmer wants to understand relationships, distributions or category differences in a dataset. For example, it can help show how values are spread, whether two variables are related, or whether one category has higher values than another. In AI programming, this supports data exploration before modelling. It helps the programmer understand the data before using it in a machine learning or deep learning workflow.

Features...

A main feature of Seaborn is that it provides attractive default styles and colour palettes. This means charts often look cleaner and more professional with less manual formatting. Seaborn also includes high-level plotting functions for common statistical charts, including distribution plots, categorical plots, regression plots, box plots, violin plots, pair plots and heatmaps. These features make it easier to create visualisations that explain patterns in data.

Another useful feature is its close integration with Pandas. Seaborn can take a DataFrame and use named columns for the x-axis, y-axis, colour grouping or category grouping. This keeps the code readable because the chart refers to meaningful column names. Seaborn can also show statistical information such as confidence intervals in some plots. Although Seaborn is built on Matplotlib, Matplotlib can still be used afterwards to adjust titles, labels, figure size and saving options. Together, Seaborn and Matplotlib provide both easy statistical plotting and detailed control.

Q2. Describe the features of good visualisation (5.3).

Good visualisation should be clear, accurate, relevant and easy to interpret. Clarity is important because the viewer should be able to understand the main message without guessing what the chart means. A clear chart has a suitable title, readable labels, sensible axis names and a layout that does not distract from the data. If the chart needs a legend, the legend should be placed where it can be read easily. The visualisation should also use the right chart type for the data. For example, a line chart is useful for showing change over time, a bar chart is useful for comparing categories, and a scatter plot is useful for showing relationships between two numerical variables.

Accuracy is another key feature. A visualisation should represent the data honestly and should not distort the values. Axis scales need to be sensible, labels need to match the data, and missing or unusual values should not be hidden in a misleading way. If a chart exaggerates differences by using a poor scale or removes important context, it can lead to the wrong conclusion. This matters in AI and data work because decisions may be made from the visual evidence.

Relevance is also important. A good visualisation should answer the question being asked and avoid unnecessary decoration. Colours, shapes and labels should support the meaning of the chart rather than make it harder to read. A chart does not need to use every possible visual feature. It should focus on the information that helps the audience understand the data. For example, if the point is to compare completion status across tasks, a simple labelled bar chart may be clearer than a complicated chart with too many colours or effects.

Accessibility and consistency are also part of good visualisation. Colours should have enough contrast, and colour choices should still work for people with colour vision difficulties. Text should be large enough to read, and important information should not rely on colour alone. In a report or evidence pack, visualisations should use consistent styling so the work looks professional. Good visualisation should also include enough context for the audience to understand what they are seeing, such as units, date ranges, categories and source information where appropriate. Overall, a good visualisation presents data in a way that is truthful, readable, relevant and useful for explaining the point being made.
