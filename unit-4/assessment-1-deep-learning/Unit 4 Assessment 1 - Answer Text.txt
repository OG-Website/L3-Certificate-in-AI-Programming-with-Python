Q1. What is deep learning?



Deep learning is a branch of artificial intelligence and machine learning that uses neural networks with multiple layers to learn patterns from data. A normal rule-based program follows instructions written by a developer, but a deep learning model learns from examples. It receives data, makes a prediction, compares that prediction with the correct answer and adjusts its internal parameters so future predictions become more accurate. The word deep refers to the layers between the input and output. Each layer transforms the data and passes a more useful representation to the next layer.



A neural network is made from connected artificial neurons. Each neuron receives input values, multiplies them by weights, adds a bias value and passes the result through an activation function. Activation functions are important because they allow the model to learn non-linear patterns rather than only simple straight-line relationships. In image work, early layers may detect edges, shapes or colour changes, while later layers combine those details into more meaningful features. In sequence work, such as price or time-series data, a model can learn how earlier values relate to later outcomes.



Deep learning is useful when the data is complex and the rules are hard to write manually. It is used in computer vision, natural language processing, speech recognition, medical imaging, recommendation systems, fraud detection and market forecasting. For example, an LSTM network can learn from sequences, which makes it suitable for time-series data. A Transformer network can use attention mechanisms to learn which parts of a sequence matter most. These models are more advanced than ordinary scripts because they learn patterns from data instead of relying only on fixed if-statements.

Q2. What is gradient descent, why is it important and what are its benefits?



Gradient descent is an optimisation method used to reduce the error of a machine learning or deep learning model. When a model is trained, it makes predictions and compares them with the correct answers. The difference is measured by a loss function. Gradient descent is the process of adjusting the model's weights so that the loss becomes smaller over time. It works by calculating the gradient, which shows the direction in which the loss increases most steeply, then moving the model parameters in the opposite direction.



A simple way to explain gradient descent is to imagine trying to walk down a hill in fog. You cannot see the whole landscape, but you can feel the slope beneath your feet. If the ground rises in one direction, you step the other way. After each step, you check the slope again and keep moving until you reach a lower point. In deep learning, the hill is the loss landscape, the current position is the model's current set of weights, and each step is a training update.



Gradient descent is important because deep learning models can contain thousands, millions or billions of parameters. It is not possible for a programmer to manually choose the best value for every weight. The model needs a mathematical process for improving itself from data. Gradient descent provides that process. It uses derivatives to work out how each weight affects the loss, then updates the weights through an optimiser. The learning rate controls the size of each update. If it is too high, the model may overshoot the best solution. If it is too low, training can be slow.



The benefits are practical. First, gradient descent gives a structured way to reduce error. Second, it scales to large datasets and complex neural networks. Third, it produces evidence that can be measured, such as training loss and validation loss over epochs. Fourth, it supports modern optimisers such as Adam, RMSprop and AdaGrad, which build on the same basic idea but improve the way updates are handled.

Q3. Give a general description of how calculus supports deep learning. Ensure you cover backpropagation.



Calculus supports deep learning because it provides the mathematics for measuring change. A deep learning model learns by changing its weights and biases during training. After the model makes a prediction, the loss function measures how wrong the prediction is. The model then needs to know how each parameter contributed to that error and how each parameter should change to reduce it. Calculus gives the training process the tools to measure those relationships.



The main calculus concept used in deep learning is the derivative. A derivative measures the rate of change of one value in relation to another. In model training, the important question is often: if this weight changes slightly, how much does the loss change? If increasing a weight increases the loss, the optimiser can reduce that weight. If changing another weight has a strong effect on the loss, the optimiser can update it more carefully.



Backpropagation is the key process that uses calculus across a neural network. During the forward pass, data moves through the layers and the model produces an output. The output is compared with the correct answer and a loss value is calculated. During the backward pass, backpropagation sends the error information backwards through the network. It uses the chain rule from calculus to calculate how much each weight and bias contributed to the final error.



The chain rule matters because a neural network is a chain of connected calculations. A weight in an early layer affects the next layer, which affects later layers, which then affects the final output. Backpropagation breaks this into smaller derivative calculations and combines them efficiently. Without this, training deep networks would be too slow and impractical because each parameter would have to be tested manually.

The LSTM model uses sequential layers, and the Transformer uses attention blocks, dense layers and normalisation. Underneath those library calls, calculus is still being used to calculate gradients, apply backpropagation and update parameters through the optimiser.

Q4. What is a derivative (slope), its purpose and why do we need them in deep learning?



A derivative is a measure of how one value changes in relation to another value. It is often described as the slope of a function at a specific point. On a straight line, the slope is constant, but on a curve the slope changes from point to point. The derivative gives the instantaneous rate of change at one point. If the derivative is positive, the function is increasing. If it is negative, the function is decreasing. If it is close to zero, the function may be flattening out.



The purpose of a derivative is to explain direction and size of change. Derivatives are used in science, engineering, economics and computing wherever one value changes because of another. In optimisation, derivatives are especially important because they help find lower or higher points on a function. In deep learning, the function being optimised is usually the loss function. The aim is to reduce the loss so the model's predictions become closer to the correct answers.



Deep learning needs derivatives because a neural network has many weights and biases. When the model gives a wrong prediction, the loss function says how wrong it was, but that is not enough by itself. The model also needs to know which weights were responsible for the error and how those weights should change. The derivative of the loss with respect to a weight tells the model how changing that weight would affect the loss. A gradient is a collection of these derivative values across many parameters.



Gradient descent uses derivatives directly. The gradient points in the direction where the loss increases most strongly. Because training wants to reduce loss, the optimiser moves the weights in the opposite direction. This is repeated many times. Derivatives therefore provide the mathematical direction for learning. Without them, the model would be changing weights randomly or relying on guesswork, which would not scale to modern deep networks.



Derivatives are also needed for backpropagation. Backpropagation calculates gradients layer by layer by sending error information backwards through the neural network. It uses the chain rule to work out how the final loss depends on each earlier layer. This allows every part of the network to improve, not only the final output layer.
