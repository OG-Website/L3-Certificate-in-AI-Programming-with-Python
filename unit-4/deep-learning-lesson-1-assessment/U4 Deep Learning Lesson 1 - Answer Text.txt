Unit A/650/0763 - AC 1.1 Explain what deep learning is.

Deep learning is a type of artificial intelligence that uses layered neural networks to learn patterns from data. A normal program follows rules written by a programmer, but a deep learning model adjusts its own internal values during training so that it can improve its predictions. The word deep refers to the use of multiple layers. Each layer transforms the data in some way, and later layers can learn more complex patterns from the earlier layers.

For example, a deep learning model could be trained to recognise images, understand text, classify data, make predictions or detect patterns that are difficult to write as fixed rules. In my OG Web.site evidence workflow, deep learning could be understood as the type of model that might learn from structured evidence data, screenshots, labels or completion records if those were prepared as training data. Unit 3 helped build the foundation for this because NumPy was used for numerical arrays and Pandas was used for structured table data. Deep learning builds on those ideas by using tensors, weights, errors and training steps to learn from data.

Unit A/650/0763 - AC 1.2 Outline how calculus supports deep learning.

Calculus supports deep learning because it provides the mathematics used to understand change. During training, a neural network makes a prediction, compares that prediction with the correct answer, and calculates an error or loss. The model then needs to know how to change its weights so that the error becomes smaller. Calculus helps with this because it can describe how the loss changes when a weight changes.

The main calculus idea used in deep learning is differentiation. Differentiation is used to calculate gradients, which show the direction and rate of change. A gradient tells the training process whether increasing or decreasing a weight is likely to reduce the loss. This is important because deep learning models can have many weights, so the model needs a systematic way to improve them.

In practical Python AI programming, libraries such as TensorFlow or PyTorch can calculate gradients automatically. Even when the library does the calculation, the idea is still based on calculus. In an OG Web.site style evidence pack, I would not only show that a model trained, but also explain that calculus supports the training process by helping the model reduce error over repeated steps.

Unit A/650/0763 - AC 2.1 Explain what a derivative is and its purpose.

A derivative is a calculation that shows how one value changes in relation to another value. It measures the rate of change of a function. For a simple example, if a graph shows distance over time, the derivative can show speed because speed is the rate at which distance changes. In machine learning and deep learning, the derivative is used to understand how a model's error changes when one of the model's weights changes.

The purpose of a derivative is to give direction and size of change. If the derivative is positive, increasing the input may increase the output. If it is negative, increasing the input may decrease the output. If it is close to zero, the output may not be changing much at that point. This is useful when trying to find a minimum value, such as the lowest possible loss during model training.

In deep learning, derivatives help the training process decide how to update the model. The model does not guess randomly. It uses derivatives to work out which way to move the weights. This makes derivatives a practical tool for improving predictions and reducing mistakes.

Unit A/650/0763 - AC 2.2 Outline the need for derivatives in deep learning.

Derivatives are needed in deep learning because a model must learn how to reduce its error. A neural network starts with weights that are not yet correct. It makes predictions, calculates a loss, and then updates the weights. To update the weights properly, the training process needs to know how each weight affects the loss. Derivatives provide that information.

Without derivatives, the model would not have a clear mathematical direction for improvement. It might change weights randomly or inefficiently, which could make training slow or unsuccessful. Derivatives allow the model to calculate gradients, and gradients are used by optimisation methods such as gradient descent. This is how the model knows whether a weight should be increased or decreased.

This matters in AI programming because deep learning models can contain thousands, millions or even billions of parameters. Manually checking each possible weight change would be impossible. Derivatives make the process systematic. In Python, deep learning libraries usually calculate these derivatives through backpropagation and automatic differentiation. The programmer may not calculate every derivative by hand, but the training process depends on them. They are one of the reasons a model can learn from data instead of only following fixed instructions.

Unit D/650/0764 - AC 2.1 Outline what gradient descent is.

Gradient descent is an optimisation method used to reduce the loss or error of a model. It works by calculating the gradient of the loss function and then moving the model's weights in the direction that should reduce the loss. The gradient points towards the direction of steepest increase, so gradient descent moves in the opposite direction to go downhill towards a lower error value.

A simple way to understand gradient descent is to imagine standing on a hill and trying to reach the lowest point. If the slope shows which way goes up, then moving against the slope takes you down. In deep learning, the hill is the loss function and the position on the hill is the current set of model weights. Each training step updates the weights slightly.

The size of each step is controlled by the learning rate. If the learning rate is too large, the model may overshoot the best point. If it is too small, training may be very slow. Gradient descent is important because it gives the model a repeatable way to improve from its mistakes. In Python, this process is usually handled by an optimiser inside a deep learning library.

Unit D/650/0764 - AC 2.2 Explain the importance of implementing gradient descent.

Implementing gradient descent is important because it is the process that allows many machine learning and deep learning models to train. A model begins with weights that are not yet useful. It then makes predictions, calculates the loss, calculates gradients and updates the weights. Gradient descent is the method that turns the gradient information into actual improvement.

If gradient descent is not implemented properly, the model may not learn. A learning rate that is too high can make training unstable because the model jumps past the best values. A learning rate that is too low can make training take too long. Poor implementation can also lead to the model getting stuck, training slowly or failing to reduce the loss. This is why gradient descent is not just a theory topic; it affects whether the model works in practice.

In an OG Web.site evidence approach, implementing gradient descent properly would mean showing training results clearly, such as loss reducing across epochs, screenshots of the notebook output, and code that explains the optimiser settings. The evidence would need to show that the model is actually learning, not just that code has been written.

Unit D/650/0764 - AC 2.3 Describe the benefits of gradient descent.

One benefit of gradient descent is that it gives a model a structured way to reduce error. Instead of changing weights randomly, the model uses the gradient to make informed updates. This makes training more efficient and gives the model a clear direction for improvement.

A second benefit is that gradient descent can work with very large models and datasets. Deep learning models often have many parameters, so it would not be practical to manually test every possible combination of weights. Gradient descent allows the model to improve step by step, even when the full problem is too large to solve by hand.

A third benefit is that gradient descent supports repeated improvement. Each training step can reduce the loss, and this can be tracked over time. In a Python notebook, this could be evidenced by showing loss values across epochs or by plotting a training curve. This would fit the OG Web.site evidence style because the output, chart and explanation could be included in a branded evidence document with GitHub links.

Gradient descent is also flexible. Different versions, such as batch gradient descent, stochastic gradient descent and mini-batch gradient descent, can be used depending on the size of the data and the training requirements. Overall, gradient descent is beneficial because it makes deep learning training possible, measurable and improvable.
